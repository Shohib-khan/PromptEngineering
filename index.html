Weeks 1-2Foundation

Mindset, setup, and AI literacy

Understand what Gen AI engineers actually do: build useful software around models, data, tools, workflows, and evaluation.

  • Install VS Code, Git, Python, Node.js, and create GitHub profile.
  • Learn terminal basics, folders, virtual environments, APIs, JSON, HTTP, and environment variables.
  • Study AI terms: model, token, context window, embedding, vector database, hallucination, latency, cost.
  • Daily task: read one AI product case study and rewrite it as a business workflow.
Deliverable: a GitHub repo named gen-ai-learning-log with daily notes, setup screenshots, and a simple API call script.
Weeks 3-8Foundation

Python, data handling, and software basics

Become comfortable enough to build backend logic without copying every line. This is where career switchers must slow down and get solid.

  • Python: variables, functions, classes, file handling, exceptions, list/dict comprehensions.
  • Data: pandas, CSV/Excel/JSON, text cleaning, basic SQL joins, aggregation, and filtering.
  • Software: Git branches, README files, modular code, logging, config files, and basic tests.
  • Mini builds: resume parser, invoice extractor, FAQ cleaner, CSV insight generator.
Deliverable: a command-line Python app that reads business documents and produces structured JSON output.
Weeks 9-12Foundation

Machine learning basics without drowning in math

You do not need to become a research scientist first, but you must understand enough ML to reason about data, metrics, and model behavior.

  • Core ideas: train/test split, features, labels, overfitting, classification, regression, clustering.
  • Metrics: accuracy, precision, recall, F1, confusion matrix, ROC basics.
  • Practice: scikit-learn pipelines and model comparison on tabular datasets.
  • AI judgment: when not to use Gen AI, when rules or classic ML are better.
Deliverable: a notebook explaining a business prediction model in simple language with metrics and limitations.
Weeks 13-17Build

NLP and LLM fundamentals

Learn how text becomes model input, how transformers changed NLP, and how modern LLM APIs are used in applications.

  • NLP: tokenization, stemming, lemmatization, embeddings, semantic similarity, chunking.
  • LLMs: transformer intuition, attention, pretraining, instruction tuning, RLHF, model limitations.
  • APIs: chat completion patterns, streaming responses, structured outputs, function calling basics.
  • Practice: summarizer, sentiment analyzer, meeting notes generator, customer email classifier.
Deliverable: a small web app that accepts long text and returns summary, key decisions, action items, and risks.
Weeks 18-20Build

Prompt engineering and evaluation

Prompting is not magic wording. It is task design, context design, output contracts, tests, and iteration.

  • Patterns: role, context, examples, constraints, rubrics, chain-of-thought alternatives, self-checking.
  • Output control: JSON schemas, validation, retry logic, guardrails, refusal handling.
  • Evaluation: golden datasets, human review, consistency tests, hallucination checks, cost tracking.
  • Practice: create 30 prompts for one business domain and measure output quality.
Deliverable: prompt evaluation dashboard showing pass/fail cases, score reasons, latency, and cost.
Weeks 21-28Advanced

RAG: retrieval augmented generation

This is one of the most employable Gen AI skills. Learn to connect private knowledge to LLMs reliably.

  • Ingestion: document loaders, chunking strategy, metadata, deduplication, cleaning.
  • Retrieval: embeddings, vector search, hybrid search, reranking, query rewriting.
  • Generation: citations, answer grounding, fallback responses, context compression.
  • Evaluation: retrieval precision, answer faithfulness, source coverage, regression tests.
Deliverable: a production-style policy/document assistant with citations, upload flow, and evaluation report.
Weeks 29-32Advanced

Agents, tools, and workflow automation

Agents are useful when they can plan, call tools, observe results, and recover. Learn the engineering discipline behind them.

  • Tool calling: database lookup, email draft, CRM update, web search, calculator, file operations.
  • Patterns: planner-executor, router, evaluator-optimizer, human-in-the-loop approvals.
  • Safety: permissions, audit logs, sandboxing, rate limits, and rollback paths.
  • Practice: automate one real workflow from your old industry experience.
Deliverable: an agentic workflow that researches a customer issue, drafts a response, and asks for approval before action.
Weeks 33-36Advanced

Production Gen AI engineering

Hiring teams care whether your app survives real users, bad inputs, changing data, and cost pressure.

  • Backend: FastAPI or Node.js, auth basics, queues, caching, background jobs.
  • Deployment: Docker basics, environment secrets, cloud deploy, monitoring, logs.
  • Quality: unit tests, integration tests, eval tests, red-team prompts, observability.
  • Responsible AI: privacy, PII handling, explainability, model risk, legal escalation.
Deliverable: deploy one Gen AI app publicly with README, architecture diagram, test results, and demo video.
Weeks 37-39Career

Portfolio that proves employability

Your portfolio should feel like work you could do on Monday morning for a company, not toy demos only.

  • Project 1: domain-specific RAG assistant with citations and evals.
  • Project 2: AI workflow automation with tool calls and approval gates.
  • Project 3: structured extraction app for invoices, resumes, claims, or support tickets.
  • Project 4: analytics assistant that answers business questions from SQL or CSV data.
Deliverable: portfolio site with live demos, GitHub links, architecture notes, business value, and limitations.
Week 40+Career

Crack Gen AI interviews and make the switch

Position yourself as a domain-aware AI application engineer, not as someone merely following tutorials.

  • Resume: translate old experience into AI workflows, automation, metrics, and user impact.
  • Interview: explain RAG, embeddings, chunking, hallucination mitigation, model selection, and eval design.
  • System design: design an enterprise knowledge assistant, ticket triage bot, or AI analyst.
  • Job strategy: target Gen AI engineer, AI application developer, RAG engineer, prompt/evaluation engineer.
Deliverable: 2-minute project pitch, 6 STAR stories, 50 interview answers, and weekly application tracker.
